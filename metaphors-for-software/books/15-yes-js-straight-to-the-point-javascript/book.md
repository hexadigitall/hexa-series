## Part 1: The Metaphorical Story

### The Story of the Mechanic Who Traveled Light

Nia is the fastest mechanic in a crowded workshop district. Her secret is not speed. It is selection. She carries only the tools that solve today's machines.

Old tools still hang on the wall, but she does not pack them for field calls. She keeps a compact kit: one wrench for constants, one for changeable parts, one folding guide for fast disassembly, and one communication radio for remote diagnostics.

At midday, another mechanic arrives with a giant bag and loses time searching for the right part. Nia opens three tools, fixes six machines, and leaves with clean notes for the next shift.

Her lesson spreads: mastery is not owning every tool ever made. Mastery is choosing the smallest reliable set that matches the work.

## Part 3: The Code Adapter

### Metaphor-to-Code Table

### Canonical Pattern (JavaScript)

- Practical ES modules layout
- Data transformation with map/filter/reduce
- API request flow with fetch and async/await
- Error handling with try/catch and fallback states

### Antipattern

- Using var in modern codebases
- Mutating shared objects accidentally with shallow copies
- Chaining promise callbacks without structured error flow

### Before / After

Before: Verbose mixed-style JavaScript with hard-to-track state.

After: Consistent ES6+ style with clear module boundaries and predictable async handling.
