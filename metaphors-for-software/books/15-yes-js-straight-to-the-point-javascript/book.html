    Part 1: The Metaphorical Story

    The Story of the Mechanic Who Traveled Light

    Nia is the fastest mechanic in a crowded workshop district. Her secret is not speed. It is selection.

    She carries only the tools that solve today's machines. Old tools still hang on the wall. Some of them work. A few of them are irreplaceable in specific, ancient situations. But Nia does not pack them for field calls. She has learned which problems repeat, and she carries the tools that solve those problems without friction.

    Her kit: one wrench for things that will not change (she has labeled it permanent), one for things that must flex throughout the shift (adjustable), one folding disassembly guide that collapses a complex object down to its components in a single motion, and a compact radio for sending requests out and reading what comes back.

    At midday, another mechanic arrives from the old school. His bag is large. His tools rattle. He spends eight minutes finding the wrench he needs. By the time he starts, Nia has already fixed three jobs and is writing notes for the next shift.

    A junior apprentice asks Nia what the trick is.

    She holds up the wrench labeled permanent. "If it should never change, use this. The system will remind you loudly if you try to swap it."

    She holds up the adjustable. "If it needs to change as the job moves, use this one. Never use the permanent wrench for it."

    She folds out the disassembly guide. "When a machine arrives, I don't carry the whole machine. I pull only the parts I need. One motion."

    She taps the radio. "And when I need something from somewhere else, I send a clean request, wait for the reply, and handle it when it arrives. I don't stop working while I wait."

    Mastery is not owning every tool ever made. Mastery is carrying the smallest reliable set that matches today's work.

    Canonical Patterns

    const and let — never var

    // const: the binding will not change
const API_URL = "https://api.hexadigitall.com/v1";

// let: the value will change inside this scope
let retries = 0;
while (retries < 3) {
  retries++;
}

// var is function-scoped and hoisted — creates unpredictable bugs
// Never use var in modern JavaScript.

    Arrow Functions

    // Traditional
function double(n) { return n * 2; }

// Arrow — concise, no own `this`
const double = (n) => n * 2;

// With a block body when you need multiple statements
const greet = (name) => {
  const msg = `Hello, ${name}`;
  return msg;
};

    Template Literals

    const name = "Nia";
const role = "mechanic";

// Old: "Hello, " + name + ". You are a " + role + "."
// New:
const intro = `Hello, ${name}. You are a ${role}.`;

    Destructuring

    // Object destructuring — pull only what you need
const tool = { name: "wrench", size: "large", material: "steel" };
const { name, size } = tool;

// Array destructuring
const [first, second] = ["A", "B", "C"];

// Destructuring with rename
const { name: toolName } = tool;

// Destructuring in function parameters
function describe({ name, size }) {
  return `${name} is ${size}`;
}

    Spread and Rest

    // Spread: expand an iterable
const tools = ["wrench", "screwdriver"];
const fullKit = [...tools, "hammer", "level"];

// Spread to shallow-copy an object
const base = { host: "localhost", port: 3000 };
const overrides = { ...base, port: 8080 };

// Rest: gather remaining arguments
function logAll(first, ...rest) {
  console.log(first, rest);
}

    Array Methods: map, filter, reduce

    const prices = [10, 25, 8, 40, 15];

// map: transform each item
const withTax = prices.map((p) => p * 1.2);

// filter: keep items that pass a test
const affordable = prices.filter((p) => p <= 20);

// reduce: collapse to a single value
const total = prices.reduce((sum, p) => sum + p, 0);

    ES Modules

    // utils.js
export function formatPrice(n) {
  return `$${n.toFixed(2)}`;
}
export const TAX_RATE = 1.2;

// main.js
import { formatPrice, TAX_RATE } from "./utils.js";
console.log(formatPrice(10 * TAX_RATE));

    Optional Chaining

    const user = { profile: { email: "nia@hex.com" } };

// Without optional chaining — throws if profile is undefined
const email = user.profile.email;

// With optional chaining — returns undefined instead of throwing
const email = user?.profile?.email;

// Combined with nullish coalescing for a clean fallback
const display = user?.profile?.email ?? "no email on file";

    Fetch with async/await

    async function getUser(id) {
  try {
    const res = await fetch(`https://api.hexadigitall.com/users/${id}`);
    if (!res.ok) throw new Error(`HTTP error: ${res.status}`);
    const data = await res.json();
    return data;
  } catch (err) {
    console.error("Fetch failed:", err.message);
    return null;
  }
}

    Antipatterns

    Mutating Shared State via Shallow Copy

    // WRONG: both variables point to the same object
const original = { host: "prod", port: 443 };
const copy = original;
copy.port = 8080;
console.log(original.port); // 8080 — original is mutated

// FIX: use spread to create a new object
const copy = { ...original };
copy.port = 8080;
console.log(original.port); // 443 — original is safe

    Callback Chain Instead of async/await

    // PROBLEMATIC: nested callbacks create hard-to-follow error paths
fetch(url1, (res1) => {
  fetch(url2, (res2) => {
    fetch(url3, (res3) => {
      // three levels deep and growing
    });
  });
});

// FIX: async/await reads like synchronous code
const res1 = await fetch(url1);
const res2 = await fetch(url2);
const res3 = await fetch(url3);

    Before and After

    Before: Mixed var, string concatenation, manual loops over arrays, nested callbacks, and manually constructed URLs.

    After: Consistent const/let, template literals, map/filter/reduce pipelines, named module boundaries, and clean async/await request flows.

    Appendix A: Lab Solutions

    Lab 15-01 — const and let

    Hint

    Use const for anything that is assigned once and does not change. Use let only when the value must be reassigned.

    Full Solution

    // Before
var count = 0;
var name = "Nia";
var items = [];
for (var i = 0; i < 3; i++) {
  items.push(i);
  count++;
}

// After
let count = 0;          // reassigned each loop
const name = "Nia";    // never reassigned
const items = [];       // array reference is constant; contents can change
for (let i = 0; i < 3; i++) {
  items.push(i);
  count++;
}

    Lab 15-02 — Array Methods

    Hint

    Chain .filter() → .map() → .reduce() in a single expression.

    Full Solution

    const nums = [3, 14, 7, 25, 11, 8];
const result = nums
  .filter((n) => n > 10)      // [14, 25, 11]
  .map((n) => n * 2)           // [28, 50, 22]
  .reduce((sum, n) => sum + n, 0); // 100
console.log(result); // 100

    Lab 15-03 — Destructuring

    Hint

    Nest the destructuring to match the object shape. Use = "default" for the fallback.

    Full Solution

    const user = {
  name: "Nia",
  address: { city: "Lagos", country: "Nigeria" }
};

const { address: { city, postcode = "N/A" } } = user;
console.log(city);     // "Lagos"
console.log(postcode); // "N/A"

    Lab 15-04 — Fetch

    Hint

    Check res.ok before calling res.json(). Wrap in try/catch.

    Full Solution

    async function fetchUser(id) {
  try {
    const res = await fetch(
      `https://jsonplaceholder.typicode.com/users/${id}`
    );
    if (!res.ok) throw new Error(`HTTP ${res.status}`);
    const { name, email } = await res.json();
    console.log(`${name} — ${email}`);
  } catch (err) {
    console.error("Failed:", err.message);
  }
}

fetchUser(1);

    Lab 15-05 — Modules

    Full Solution

export const PI = 3.14159;
