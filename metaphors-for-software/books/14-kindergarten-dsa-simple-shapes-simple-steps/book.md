# Kindergarten DSA: Simple Shapes, Simple Steps

**Domain:** The Workshop | **Book 14**

**Metaphor:** A playground where every game teaches one core structure or algorithm

**Technical Goal:** Remove fear from DSA by replacing jargon-first teaching with shape-first intuition

---

## Part 1: The Metaphorical Story

### The Story of Bit and the Playground Map

Bit is the playground helper robot. Every morning starts with one mission: organize the day before recess chaos begins.

In one corner, Bit lines up cubbies in a numbered row. In another, clue cards lead from one hiding spot to the next. Pancakes pile on one tray, while children stand in a snack line that must stay fair.

When a whistle blows, Bit has to find a missing red ball. First he checks each cubby one by one. Too slow. Next time he sorts cubbies by color and number, then jumps straight to the middle and cuts the search in half.

By sunset, Bit learns the secret: the shape of the storage changes the speed of the search. Order is not decoration. Order is performance.

### Canonical Pattern (Python + JavaScript)

- Side-by-side implementations for core structures
- Search and sort walkthroughs with step counters
- Complexity annotations tied to each playground activity

### Antipattern

- Prematurely using complex structures for tiny datasets
- Ignoring input order assumptions before binary search
- Confusing recursion with looping without base cases

### Before / After

Before: Problem solving by trial-and-error loops only.

After: Deliberate structure selection with predictable time and space trade-offs.
