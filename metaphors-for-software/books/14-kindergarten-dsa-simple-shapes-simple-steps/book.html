    Part 1: The Metaphorical Story

    The Story of Bit and the Playground Map

    Bit is the playground helper robot. Every morning starts with one mission: organize the day before recess chaos begins.

    In one corner, Bit lines up cubbies in a numbered row — one slot per child, perfectly ordered. A child who wants the third cubby does not search. They count to three and reach directly in. Fast. Predictable. But the cubbies are fixed. Add a new child mid-week and the row has no room. Everything must shift.

    In another corner, Bit sets out clue cards. The first card does not hold the treasure. It holds a map to the second card. The second leads to the third. The chain is flexible — add a new clue anywhere without moving the others. But if you want the seventh clue, you must follow the chain from the start. There are no shortcuts.

    For breakfast, Bit stacks pancakes onto a single tray. The last pancake made sits on top. It is the first one eaten. The plate at the bottom waits longest. Order-in-reverse. Last in, first out. If you want the bottom pancake, you must lift every layer above it first.

    For the ice cream line, the rule is different. Whoever arrived first gets served first. No cutting. No skipping. The line is fair because it is ordered by arrival, and arrival cannot be faked.

    When a whistle blows and a red ball goes missing, Bit checks every cubby one by one. Box one. Box two. Box three. On a bad day it is the last box checked. Bit counts twenty-eight steps to find one ball.

    The next morning, Bit sorts the cubbies by ball color before play begins. Now the search is different. Jump to the middle. Is the red section left or right? Left. Jump to the middle of the left side. Is it here or further left? Here. Three steps instead of twenty-eight. The shape of the storage changed the speed of the search.

    By sunset, Bit writes one line on the chalkboard: Order is not decoration. Order is performance.

    Canonical Patterns

    Array and Linear Search (Python)

    # Array: indexed, fixed-size in concept
cubbies = ["red ball", "blue block", "puzzle", "crayon"]

# Linear search — check every slot
def find_item(cubbies, target):
    for i, item in enumerate(cubbies):
        if item == target:
            return i
    return -1

print(find_item(cubbies, "puzzle"))  # 2

    Binary Search (Python) — requires sorted input

    def binary_search(sorted_items, target):
    low, high = 0, len(sorted_items) - 1
    while low <= high:
        mid = (low + high) // 2
        if sorted_items[mid] == target:
            return mid
        elif sorted_items[mid] < target:
            low = mid + 1
        else:
            high = mid - 1
    return -1

    Stack (Python)

    stack = []
stack.append("pancake 1")   # push
stack.append("pancake 2")
stack.append("pancake 3")
print(stack.pop())           # "pancake 3" — last in, first out

    Queue (Python)

    from collections import deque
queue = deque()
queue.append("first child")   # enqueue
queue.append("second child")
print(queue.popleft())         # "first child" — first in, first out

    Hash Map (Python)

    # O(1) average lookup — no searching required
lockers = {
    "Bit": 42,
    "Mara": 17,
    "Nia": 8,
}
print(lockers["Bit"])  # 42 — directly

    Same Structures in JavaScript

    // Array
const cubbies = ["red ball", "blue block", "puzzle"];

// Stack via Array
const stack = [];
stack.push("pancake 1");
stack.push("pancake 2");
console.log(stack.pop());  // "pancake 2"

// Queue via Array (not optimal for large data — use a linked structure)
const queue = [];
queue.push("first child");
queue.push("second child");
console.log(queue.shift()); // "first child"

// Hash Map
const lockers = new Map();
lockers.set("Bit", 42);
console.log(lockers.get("Bit")); // 42

    Antipatterns

    Binary Search on Unsorted Data

    # WRONG: binary search on unsorted list returns wrong results silently
items = [7, 1, 9, 3, 5]
result = binary_search(items, 3)  # may return -1 incorrectly
# FIX: always sort before binary searching
items.sort()
result = binary_search(items, 3)  # correct

    Array for High-Frequency Middle Insertions

    # O(n) cost per insert — every item after the insertion point must shift
items = list(range(1_000_000))
items.insert(500_000, 99)  # slow: half a million shifts

# FIX: if you need many middle insertions, a linked list or deque is cheaper
from collections import deque
d = deque(range(1_000_000))
d.insert(500_000, 99)  # still O(n) for deque insert, but better for front/back ops

    Before and After

    Before: Solving every problem with a flat list and a loop, regardless of how often items are searched, inserted, or removed.

    After: Choosing the structure based on the dominant operation — array for random access, linked list for frequent insertion, stack for LIFO unwinding, queue for ordered processing, hash map for fast lookup.

    Appendix A: Lab Solutions

    Lab 14-01 — Array Basics

    Hint

    Iterate with a counter. Return the index when the item matches, or -1 at the end.

    Step by Step

    1. "apple" at index 0 — no match.
    2. "banana" at index 1 — no match.
    3. "cherry" at index 2 — match. 3 comparisons.

    Full Solution

    def linear_search(items, target):
    for i, item in enumerate(items):
        if item == target:
            return i, i + 1  # index, steps
    return -1, len(items)

items = ["apple", "banana", "cherry", "date"]
index, steps = linear_search(items, "cherry")
print(f"Found at index {index} in {steps} step(s)")

    Lab 14-02 — Binary Search

    Hint

    Sort first. Then check the midpoint and halve the search space each round.

    Step by Step

    1. Sorted: [1, 3, 7, 8, 15, 20]
    2. Mid index 2 = 7. Target 8 is greater. Search right half.
    3. Mid index 4 = 15. Target 8 is smaller. Search left.
    4. Mid index 3 = 8. Found. 3 comparisons.

    Full Solution

    nums = [15, 3, 8, 1, 20, 7]
nums.sort()  # [1, 3, 7, 8, 15, 20]
steps = 0
low, high = 0, len(nums) - 1
target = 8
while low <= high:
    steps += 1
    mid = (low + high) // 2
    if nums[mid] == target:
        print(f"Found at index {mid} in {steps} step(s)")
        break
    elif nums[mid] < target:
        low = mid + 1
    else:
        high = mid - 1

    Lab 14-03 — Stack vs Queue

    Hint

    Stack removes from the back (LIFO). Queue removes from the front (FIFO).

    Full Solution

    from collections import deque

items = ["A", "B", "C", "D"]

# Stack output: D C B A
stack = []
for i in items: stack.append(i)
while stack: print(stack.pop(), end=" ")

# Queue output: A B C D
queue = deque(items)
while queue: print(queue.popleft(), end=" ")

    Lab 14-04 — Hash Map Lookup

    Hint

    Use a dictionary. dict.get(key, default) returns the default if the key is absent.

    Full Solution

    playground = {
    "ball": "red rubber ball",
    "cube": "foam building cube",
    "rope": "jump rope 3m",
    "ring": "hula hoop",
    "bat":  "foam bat",
}
def lookup(item):
    return playground.get(item, "not found")

print(lookup("rope"))  # "jump rope 3m"
print(lookup("kite"))  # "not found"

    Lab 14-05 — Big O Analysis

    Full Solution

    (a) Printing every item in a list:
    One operation per item → O(n)

(b) Nested loop over the same list:
    Each of n items triggers n inner operations → O(n²)

(c) Hash map lookup:
    One direct computation regardless of size → O(1) average
