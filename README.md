# Hexa Series

**Hexadigitall Technologies — Educational Publishing**

---

## What This Repository Is

**Hexa Series** is an umbrella repository for multiple distinct educational publishing projects produced by Hexadigitall Technologies.

Each series follows its own audience, tone, format, and content architecture. They share a production philosophy: every title must be visually rich, conceptually honest, and structured for learner agency.

---

## Series in This Repository

### [hexa-ai-series/](hexa-ai-series/)

**The HexaVerse — Science, Space, and the Frontier**

An expanding educational universe that teaches physics, space engineering, AI, biology, and future-civilization design. Follows Hexa, a young AI-augmented explorer, from childhood wonder through advanced systems thinking.

- **Audience:** Ages 5 through adult
- **Arcs:** The Galactic Playground, Mission to Mars, The Silicon Astronaut, Automating the Galaxy
- **Formats:** Storybooks, course modules, labs, capstones
- **Status:** Active — first book complete, illustration in progress

→ See [hexa-ai-series/README.md](hexa-ai-series/README.md)

---

### [metaphors-for-software/](metaphors-for-software/)

**Metaphors for Software — A Physical Craft Curriculum**

A 12-book series that treats software as a physical craft. Each book opens with a 5-page metaphorical story, builds understanding through high-contrast visual diagrams, and closes by translating the metaphor into real working code.

The series follows a career path through three domains:
- **The Workshop** — The individual artisan: OOP, data structures, recursion, memory
- **The City** — Infrastructure and communication: networking, APIs, load balancing, security
- **The Factory** — DevOps and delivery: containers, IaC, CI/CD, orchestration

- **Audience:** Developers, technical learners, and curious non-programmers at all levels
- **Format:** Technical books with narrative, visual diagrams, and code
- **Status:** Active — series defined, Book 1 in development

→ See [metaphors-for-software/README.md](metaphors-for-software/README.md)

## Shared Publishing Philosophy

Across all series:

- Every title must teach through story, not only through instruction
- Visual communication is a first-class learning tool, not decoration
- Accessibility (readability, contrast, representation) is a production requirement
- Science and technical accuracy are non-negotiable
- Titles are designed to scale into print, digital, and classroom formats
