# Hexadigitall Book Release Checklist

Title: ___________________________  
Version: ___________  
Release Target Date: ___________  

Use this checklist to track readiness across all gates before publication.

---

## Gate 1: Content Completeness

- [ ] All required chapter sections present (Hook, Explain, Show, Try, Reflect, Check)
- [ ] Metadata file complete and valid JSON
- [ ] Chapter objectives mapped to learning outcomes
- [ ] Glossary source file complete with all terms
- [ ] All illustrations received from illustrator
- [ ] Illustration rights agreements signed
- [ ] Interaction specs finalized (A-Z, Ten-Word, Remix)
- [ ] Teacher guide or parent guide drafted

**Approval sign-off:** _________________ Date: _______

---

## Gate 2: Science and Pedagogy Review

- [ ] Science reviewer approved all technical claims
- [ ] Pedagogy reviewer approved learning progression
- [ ] All claims tagged with taxonomy (Established, Emerging, Speculative)
- [ ] Misconceptions addressed and noted
- [ ] Age appropriateness verified by target audience band
- [ ] Inclusive representation verified (characters, examples, perspectives)
- [ ] No fear-based framing; challenges framed as design problems
- [ ] Gamification challenges tested for learning outcome

**Science reviewer sign-off:** _________________ Date: _______

**Pedagogy reviewer sign-off:** _________________ Date: _______

---

## Gate 3: Design and Accessibility

- [ ] Visual hierarchy consistent throughout
- [ ] Arc color applied correctly (spine, headers, accent elements)
- [ ] Hexa character depicted consistently
- [ ] Secondary characters reused from canon
- [ ] All illustrations at minimum 300 DPI for print
- [ ] Color contrast checked (WCAG AA minimum)
- [ ] Alt text written for all images and diagrams
- [ ] Font sizing age-appropriate and readable
- [ ] Line length and leading optimized for reading
- [ ] Accessibility badges earned or on track
- [ ] Print mockup reviewed if print edition planned

**Design QA sign-off:** _________________ Date: _______

**Accessibility QA sign-off:** _________________ Date: _______

---

## Gate 4: IP and Legal

- [ ] All illustration rights cleared (signed agreements)
- [ ] All third-party imagery licensed or original
- [ ] Trademark elements verified (logos, colors, brand markers)
- [ ] Gamification feature names protected (patent/trademark filed or planned)
- [ ] Contributor agreements signed (author, illustrator, reviewers)
- [ ] Copyright notice accurate
- [ ] Privacy and FERPA compliant if using student work/examples
- [ ] No unlicensed character, world, or brand references

**Legal sign-off:** _________________ Date: _______

---

## Gate 5: Production Readiness (Print)

*Skip if digital-only edition*

- [ ] Print-ready PDF generated and tested
- [ ] Trim size approved
- [ ] Bleed and margins validated (0.125" minimum)
- [ ] Fonts embedded or converted to curves
- [ ] Images in CMYK or RGB (no spot colors unless approved)
- [ ] Barcode and ISBN placement approved
- [ ] Spine text readable and properly oriented
- [ ] Physical proof requested from printer
- [ ] Proof reviewed and approved
- [ ] First print batch approved for release
- [ ] Inventory reserved for launch

**Print production sign-off:** _________________ Date: _______

---

## Gate 6: Sales and Marketing Readiness

- [ ] Sales copy written and approved
- [ ] Product page drafted (with cover image, description, learning outcomes)
- [ ] One-page educator brief completed
- [ ] Parent/guardian guide summary written
- [ ] Sample chapter PDF prepared (first 15-20 pages)
- [ ] Comparison chart against adjacent titles drafted
- [ ] Social media copy prepared (3-5 posts minimum)
- [ ] Launch email sequence drafted
- [ ] Pricing and packaging finalized
- [ ] SKUs created (Student Edition, Teacher Edition, Bundle options)
- [ ] ISBN assigned if applicable

**Sales and marketing sign-off:** _________________ Date: _______

---

## Final Quality Assurance

- [ ] Book readiness scorecard completed (target: 28/35 minimum)
- [ ] All gates passed with documented approvals
- [ ] Release notes prepared
- [ ] Changelog updated in content registry
- [ ] QR code and digital companion links verified
- [ ] Cross-links to adjacent titles tested
- [ ] Final proofreading complete (no typos, consistent formatting)

**Product owner final approval:** _________________ Date: _______

---

## Launch Execution

**Phase 1: Internal Readiness**
- [ ] Product pages live on preview site
- [ ] Sample chapters available
- [ ] FAQ and support scripts ready
- [ ] Team trained on value proposition and key messaging

**Phase 2: Pilot Release**
- [ ] Educator cohort invited (target: 10-20 teachers)
- [ ] Pilot access provided (free or discounted)
- [ ] Feedback collection forms deployed
- [ ] Classroom usage monitored

**Phase 3: General Release**
- [ ] Product pages live on main site
- [ ] Press release distributed
- [ ] Partner amplification activated
- [ ] Conversion tracking enabled
- [ ] Daily sales monitoring begins
- [ ] Support team briefed and ready

---

## Post-Launch Review (30-90 days)

- [ ] Weekly sales data collected
- [ ] Conversion rates analyzed
- [ ] Customer feedback reviewed
- [ ] Educator feedback summary prepared
- [ ] Errata log initiated
- [ ] Reprint or digital update decisions made
- [ ] Lessons learned documented

**Launch review sign-off:** _________________ Date: _______
